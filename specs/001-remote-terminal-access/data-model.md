# Data Model: Remote Terminal Access from iOS

**Date**: 2026-02-11
**Branch**: `001-remote-terminal-access`

## Overview

This feature does not involve application data or a database.
The "entities" are runtime system concepts managed by tmux,
mosh, and SSH. This document maps spec entities to their
concrete system representations.

## Entities

### Terminal Session (tmux session)

A named, persistent process group managed by the tmux server.

| Attribute        | Representation                          |
|------------------|-----------------------------------------|
| Name             | tmux session name (e.g., "main")        |
| Panes/Windows    | tmux windows and panes within session   |
| Creation time    | Implicit (tmux server tracks)           |
| Attached clients | Zero or more tmux client connections    |
| Persistence      | Lives until explicitly killed or reboot |

**State transitions**:
- Created: `tmux new-session -s <name>`
- Attached: `tmux attach -t <name>` (from any client)
- Detached: Client disconnects; session continues
- Destroyed: `tmux kill-session -t <name>` or system reboot

### Remote Connection (Mosh + SSH)

An encrypted network session from an iOS device.

| Attribute         | Representation                           |
|-------------------|------------------------------------------|
| Transport         | Mosh (UDP 60000-61000) + SSH (TCP 22)    |
| Authentication    | SSH key-based (initial handshake)        |
| Connection state  | Connected / Roaming / Disconnected       |
| Resilience        | Mosh SSP auto-reconnects on IP change    |
| Terminal session  | Bound to tmux via `tmux attach` command  |

**Connection flow**:
1. iOS app initiates Mosh connection (SSH handshake first)
2. Mosh server starts on NixOS, assigns UDP port
3. User runs `tmux attach -t main` inside Mosh session
4. On network interruption: Mosh SSP resynchronizes
5. On disconnect: Mosh server exits, tmux session persists

### Local Connection (WezTerm + tmux)

A WezTerm instance running on the desktop.

| Attribute         | Representation                           |
|-------------------|------------------------------------------|
| Terminal emulator | WezTerm process                          |
| Session binding   | `default_prog` launches `tmux new -As main` |
| Coexistence       | tmux allows multiple attached clients    |

**Interaction**: WezTerm renders tmux output. Pane management
is done via tmux keybindings, not WezTerm's. WezTerm's tab
bar is disabled.

## Configuration Files (NixOS-managed)

| File                     | Purpose                            |
|--------------------------|------------------------------------|
| `remote-terminal.nix`   | NixOS module: tmux, mosh, SSH      |
| `/etc/tmux.conf`        | Generated by `programs.tmux`       |
| `~/.wezterm.lua`        | User config: WezTerm + tmux launch |

## Relationships

```text
WezTerm (desktop) ──attach──> tmux session <──attach── Mosh (iOS)
                                   │
                              tmux server
                            (persistent on NixOS)
```

Both clients attach to the same tmux session simultaneously.
Changes from either client are visible to both in real time.
